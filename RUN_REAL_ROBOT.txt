# Unitree G1 Bicep Curl - Quickstart Guide

This repository contains scripts for both MuJoCo simulation and real robot control.

================================================================================
SECTION 1: SIMULATION (Verification)
================================================================================
Use this to test your logic safely before running on the real robot.

1. START THE SIMULATOR (Terminal A):
   python3.12 ark_unitree_g1/tests/g1_mujoco_sim/unitree_mujoco.py
   # This is the MuJoCo bridge; keep it running before the controller.
   
2. RUN THE CONTROLLER (Terminal B):
   python3.12 real_bicep_curl.py --sim

================================================================================
SECTION 1B: LOCAL VOICE QUICK START
================================================================================
Use this to run the fully local voice assistant on the Linux PC.

0. INSTALL VOICE DEPENDENCIES:
   pip install openai-whisper sounddevice scipy numpy pyttsx3
   sudo apt install espeak-ng -y

1. LIST MICROPHONES (optional, if the default device is wrong):
   python3.12 brunken_voice.py --list-audio-devices

   NOTE: If PortAudio is not available on this machine, the script automatically
   falls back to ALSA `arecord` for microphone capture and listing.

2. RUN THE VOICE ASSISTANT:
   python3.12 brunken_voice.py --interface enp129s0 --domain 0

   For strong English recognition:
   python3.12 brunken_voice.py --interface enp129s0 --domain 0 --language en

3. USING THE BLUE YETI MICROPHONE (Recommended):
   If the Blue Yeti is plugged into your PC via USB, it often registers as 'hw:2,0'. Test it with:
   
   For local simulation mode (Matches Simulator Domain 1):
   python3.12 brunken_voice.py --interface lo --domain 1 --language en --arecord-device "hw:2,0"
   
   For real robot mode (robot connected via Ethernet):
   python3.12 brunken_voice.py --interface enp129s0 --domain 0 --language en --arecord-device "hw:2,0"

   *Note: If you encounter audio distortion, change "hw:2,0" to "plughw:2,0" to allow Linux to resample.*
   
5. CUSTOMIZING THE WAKE WORD (Optional):
   You can change the robot's "name" by using the `--wake-word` flag:
   `python3.12 brunken_voice.py --interface lo --domain 1 --wake-word "robot"`
   (The robot will now respond to "robot salute" instead of "cheese salute")

4. TROUBLESHOOTING MICROPHONE (if Yeti is not hw:2,0):
   Run this to find your ALSA hardware device names:
   arecord -l
   Look for "Yeti Stereo Microphone" and note the card and device numbers (e.g., card 2, device 0 = hw:2,0).

5. EXERCISE / MOTION COMMANDS:
    - Say: "cheese curl" or "cheese bicep"
       -> Robot starts a continuous bicep curl animation.
    - Say: "cheese squat"
       -> Robot starts a continuous squat loop.
    - Say: "cheese lateral raise" or "cheese shoulder raise"
       -> Robot starts a continuous lateral raise loop.
    - Say: "cheese salute"
       -> Robot performs a right-hand greeting.
    - Say: "cheese stop"
       -> Stops all movement and cancels bicep curls.

================================================================================
SECTION 2: REAL ROBOT
================================================================================
Use this ONLY after verifying logic in simulation.

1. FIX NETWORKING (Run once per session):
   # Robot cable must be connected and robot Ethernet must be powered.
   # Most Linux systems will automatically assign an IP (like 192.168.123.x) when plugged in.
   # Verify your connection by running:
   ip addr show enp129s0

   # If you see an "inet 192.168.123.x" address, you are good to go! You do not need sudo.

   If this fails with "NO-CARRIER" or "Device is not activated", check:
   - Ethernet link LEDs are on at both ends
   - Robot is powered and Ethernet service is up
   - Correct NIC is used (verify with: ip -br link)

2. RUN THE CONTROLLER:
   python3.12 real_bicep_curl.py

3. STOP ROBOT:
   Press Ctrl+C in the terminal. The script will return arms home safely.

================================================================================
FILE REFERENCE
================================================================================
- real_bicep_curl.py              : The main logic (Target: Real or Sim).
- ark_unitree_g1/.../unitree_mujoco.py : The MuJoCo window (Simulator).
- ark_unitree_g1/.../config.py     : Simulation settings (Gains, Interface, Elastic Band).

================================================================================
SAFETY REMINDERS
================================================================================
- For Real Robot: Must be in Low-level Control Mode (Debug Mode).
- For Real Robot: Use a stand or ensure a clear, safe area.
- ALWAYS test new logic in SIMULATION first using the --sim flag.

================================================================================
HOW TO CUSTOMIZE THE MOTION
================================================================================
Open 'real_bicep_curl.py' and look for these variables:

- CURL_PERIOD    : Time in seconds for 1 full curl. Smaller = Faster.
- ELBOW_EXTENDED : The arm position at the BOTTOM (Limit: -1.0).
- ELBOW_FLEXED   : The arm position at the TOP (Limit: 2.0).

NOTE ON COLLISION AVOIDANCE: 
The script now includes a 'prep_q' pose that pitches the shoulders forward 
and outward. If the arms are still too close to the legs, increase the 
negative value of `prep_q[0]` (Left Shoulder Pitch) and `prep_q[7]` 
(Right Shoulder Pitch) in the script.
